Elemento Raiz
No. Elemento Descripcion
1 < !doctype html > Representa la raíz de un documento HTML o XHTML. Todos los demás elementos deben ser descendientes de este elemento.
Metadatos del Documento
No. Elemento Descripcion
2 < head > Representa una colección de metadatos acerca del documento, incluyendo enlaces a, o definiciones de, scripts y hojas de estilo.
2 < base > Define la URL base para las URLs relativas en la página.
4 < link > Usada para enlazar JavaScript y CSS externos con el documento HTML actual.
5 < meta > Define los metadatos que no pueden ser definidos usando otro elemento HTML.
6 < style > Etiqueta de estilo usada para escribir CSS en línea.
Scripting
No. Elemento Descripcion
7 < script> Define ya sea un script interno o un enlace hacia un script externo. El lenguaje de programación es JavaScript
8 < noscript > Define un contenido alternativo a mostrar cuando el navegador no soporta scripting.
Secciones
No. Elemento Descripcion
9 < body > Representa el contenido principal de un documento HTML. Solo hay un elemento < body > en un documento.
10 < section > Define una sección en un documento.
11 < nav > Define una sección que solamente contiene enlaces de navegación
12 < article > Define contenido autónomo que podría existir independientemente del resto del contenido.
13 < aside > Define algunos contenidos vagamente relacionados con el resto del contenido de la página. Si es removido, el contenido restante seguirá teniendo sentido
14 < h1 >,< h2 >,< h3 >,< h4 >,< h5 >,< h6 > Los elemento de cabecera implementan seis niveles de cabeceras de documentos; < h1 > es la de mayor y < h6 > es la de menor importancia. Un elemento de cabecera describe brevemente el tema de la sección que introduce.
15 < header > Define la cabecera de una página o sección. Usualmente contiene un logotipo, el título del sitio Web y una tabla de navegación de contenidos.
16 < footer >
Define el pie de una página o sección. Usualmente contiene un mensaje de derechos de autoría, algunos enlaces a información legal o direcciones para dar información de retroalimentación.
17 < address > Define una sección que contiene información de contacto.
18 < main > Define el contenido principal o importante en el documento. Solamente existe un elemento
en el documento.
Agrupación de Contenido
No. Elemento Descripcion
19 < p > Define una parte que debe mostrarse como un párrafo.
20 < hr > Representa un quiebre temático entre párrafos de una sección o articulo o cualquier contenido.
21 < pre > Indica que su contenido esta preformateado y que este formato debe ser preservado.
22 < blockquote > Representa un contenido citado desde otra fuente.
23 < ol > Define una lista ordenada de artículos.
23 < li > Define un artículo de una lista enumerada.
24 < dl > Define una lista de definiciones, es decir, una lista de términos y sus definiciones asociadas.
25 < dt > Representa un término definido por el siguiente < dd >
26 < dd > Representa la definición de los términos listados antes que él.
28 < figure > Representa una figura ilustrada como parte del documento.
29 < figcaption > Representa la leyenda de una figura.
30 < div > Representa un contenedor genérico sin ningún significado especial.
Semántica a nivel de Texto
No. Elemento Descripcion
31 < a > SRepresenta un hiperenlace, enlazando a otro recurso.
32 < em > Representa un texto enfatizado, como un acento de intensidad.
33 < strong > Representa un texto especialmente importante.
34 < small > Representa un comentario aparte, es decir, textos como un descargo de responsabilidad o una nota de derechos de autoría, que no son esenciales para la comprensión del documento.
35 < s > Representa contenido que ya no es exacto o relevante.
36 < cite > Representa el título de una obra.
37 < q > Representa una cita textual inline.
38 < dfn > Representa un término cuya definición está contenida en su contenido ancestro más próximo.
39 < abbr > Representa una abreviación o un acrónimo ; la expansión de la abreviatura puede ser representada por el atributo title.
40 < data > Asocia un equivalente legible por máquina a sus contenidos. (Este elemento está sólamente en la versión de la WHATWG del estandar HTML, y no en la versión de la W3C de HTML5).
41 < Time > Representa un valor de fecha y hora; el equivalente legible por máquina puede ser representado en el atributo datetime.
42 < code > Representa un código de ordenador.
43 < var> Representa a una variable, es decir, una expresión matemática o contexto de programación, un identificador que represente a una constante, un símbolo que identifica una cantidad física, un parámetro de una función o un marcador de posición en prosa.
44 < sapm > Representa la salida de un programa o un ordenador.
45 < kbd > Representa la entrada de usuario, por lo general desde un teclado, pero no necesariamente, este puede representar otras formas de entrada de usuario, como comandos de voz transcritos.
46 < sub > Representa subíndice
47 < sup > Representan superíndice
48 < i > Representa un texto en una voz o estado de ánimo alterno, o por lo menos de diferente calidad, como una designación taxonómica, un término técnico, una frase idiomática, un pensamiento o el nombre de un barco.
49 < b > Representa un texto hacia el cual se llama la atención para propósitos utilitarios. No confiere ninguna importancia adicional y no implica una voz alterna.
50 < u > Representa una anotación no textual sin-articular, como etiquetar un texto como mal escrito o etiquetar un nombre propio en texto en chino.
51 < mark > Representa texto resaltado con propósitos de referencia, es decir por su relevancia en otro contexto.
52 < ruby > Representa contenidos a ser marcados con anotaciones ruby, recorridos cortos de texto presentados junto al texto. Estos son utilizados con regularidad en conjunto a lenguajes de Asia del Este, donde las anotaciones actúan como una guía para la pronunciación, como el furigana japonés.
53 < rt > Representa el texto de una anotación ruby
54 < rp > Representa los paréntesis alrededor de una anotación ruby, usada para mostrar la anotación de manera alterna por los navegadores que no soporten despliegue estandar para las anotaciones.
55 < dbi > Representa un texto que debe ser aislado de sus alrededores para el formateado bidireccional del texto. Permite incrustar un fragmento de texto con una direccionalidad diferente o desconocida.
56 < bdo > Representa la direccionalidad de sus descendientes con el fin de anular de forma explícita al algoritmo bidireccional Unicode.
57 < span > Representa texto sin un significado específico. Este debe ser usado cuando ningún otro elemento semántico le confiere un significado adecuado, en cuyo caso, provendrá de atributos globales como class, lang, o dir.
58 < br > Representa un salto de línea.
59 < wbr > Representa una oportunidad de salto de línea, es decir, un punto sugerido de envoltura donde el texto de múltiples líneas puede ser dividido para mejorar su legibilidad.
